# Code Of Conduct

Premature Contribution Firewall is built for maintainers under pressure. The project should model the same standards it asks contributors to meet: clarity, evidence, restraint, and respect for review time.

## Expected Behavior

- Be direct without being abusive.
- Critique claims, evidence, and patches rather than people.
- Keep feedback actionable and specific.
- Assume maintainers have limited time.
- Separate proven facts from guesses.
- Respect private data, security boundaries, and maintainer feedback.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publishing private repository data, credentials, or maintainer feedback without permission.
- Pressuring maintainers to review broad or unverified work.
- Using the project to shame contributors for using tools or assistants.
- Misrepresenting PCF as a tool for identifying who or what wrote a contribution.

## Enforcement

Maintainers may edit, hide, close, or lock issues and pull requests that violate these expectations. Repeated or severe violations can lead to a ban from project spaces.

If a conduct concern involves security-sensitive material, follow [SECURITY.md](SECURITY.md) and avoid posting details publicly.
